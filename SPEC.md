# ТЗ: программа «Стикер»

Конвертер медиафайлов в стикеры Telegram-формата: **WebM VP9, 512×512, ≤ 256 КБ, ≤ 3 с, без аудио**.
Платформы: Windows (этап 1), Android (этап 2). Требование: портативность, минимальный размер дистрибутива.

---

## 1. Стек технологий

### Рекомендуемый: Tauri 2 (Rust + HTML/JS UI) + кастомная сборка FFmpeg

| Компонент | Технология | Размер | Обоснование |
|---|---|---|---|
| Ядро конвертации | FFmpeg (минимальная сборка) | ~15–25 МБ | Единственный инструмент, покрывающий все форматы + libvpx-vp9 + двухпроходное кодирование под целевой размер |
| GUI + оркестрация | Tauri 2 (Rust) | ~8–12 МБ exe | Портативный один exe, WebView2 предустановлен в Win10/11 (рантайм не тащим) |
| Предпросмотр видео | `<video>` в WebView2 (Chromium) | 0 МБ | Chromium нативно играет WebM/VP9, MP4/H.264, GIF — плеер не нужен |
| Android (этап 2) | Tauri 2 Android + тот же UI | — | Один код UI и алгоритмов на обе платформы; ffmpeg шипуется как `libffmpeg.so` в jniLibs и запускается exec'ом (стандартный приём, работает на Android 10+) |

**Итог дистрибутива Windows: ~25–35 МБ, одна папка, без установки и админ-прав.**

Кастомная сборка FFmpeg (вместо готовой essentials ~80 МБ):
- Демуксеры: mov/mp4, matroska/webm, gif, image2, apng, webp_pipe
- Декодеры: h264, hevc, vp8, vp9, av1 (dav1d), mjpeg, png, gif, webp, bmp
- Энкодер: только libvpx-vp9
- Фильтры: scale, crop, pad, fps, trim, setpts, format
- Аудио полностью отключено (`--disable-avdevice`, без аудиокодеков) — стикеры без звука
- Fallback на первом этапе: готовая сборка gyan.dev essentials, замена на кастомную позже

### Альтернатива (если Rust — блокер): C# WinForms на .NET Framework 4.8
exe ~2 МБ без рантайма (4.8 предустановлен в Win10/11). Минус: предпросмотр WebM придётся делать покадровой отрисовкой через ffmpeg-pipe, и для Android нужна вторая кодовая база (Kotlin). Выбирать только если Windows-версия сильно приоритетнее кроссплатформенности.

## 2. Жёсткие ограничения выходного файла

| Параметр | Значение | Контроль |
|---|---|---|
| Контейнер/кодек | WebM / VP9 (по умолчанию) | параметры энкодера |
| Разрешение | 512×512 (по умолчанию) | фильтр scale/crop |
| Размер файла | **≤ 256 КБ (262 144 байт), строго** | итеративный алгоритм (п. 4) + проверка размера после кодирования |
| Длительность | **≤ 3.000 с, строго** | обрезка до 3.0 с; после кодирования проверка через ffprobe; при превышении из-за округления контейнера — пережим с длительностью 2.97 с |
| FPS | ≤ 30 | фильтр fps |
| Аудио | отсутствует | `-an` |

## 3. Функциональность по типам входных файлов

Поддерживаемые входы: JPG, PNG, WebP, BMP (статика); GIF, анимированный WebP, APNG (анимация); MP4, MOV, MKV, WebM, AVI (видео).

| Функция | Фото | GIF / аним. | Видео |
|---|---|---|---|
| Обрезка длительности (по умолч. 0–3 с) | — | ✓ | ✓ |
| Масштабирование до 512×512 | ✓ | ✓ | ✓ |
| Режим масштабирования: растянуть (умолч.) / увеличить и обрезать | ✓ | ✓ | ✓ |
| Ограничение FPS до 30 | — | ✓ | ✓ |
| Удаление аудиодорожки | — | — | ✓ |
| Подгон под 256 КБ | тривиально (1 кадр, CRF) | ✓ | ✓ |
| Предпросмотр с таймлайном | статичная картинка | ✓ | ✓ |

Элементы UI, неприменимые к типу файла, скрываются (у фото нет слайдера длительности).

Режимы масштабирования:
- **Растянуть** (умолч.): `scale=512:512` — искажение пропорций допускается.
- **Увеличить и обрезать** (cover): `scale=512:512:force_original_aspect_ratio=increase, crop=512:512` — без искажений, края обрезаются по центру.

Фото кодируется как WebM из одного кадра (`-t 0.1`, CRF-режим) — размер почти всегда сразу в лимите.

## 4. Алгоритм подгона под 256 КБ (ядро программы)

Двухпроходное VBR-кодирование libvpx-vp9 с целевым битрейтом + итеративное снижение при промахе.

```
TARGET = 254 KB                      # запас 2 КБ на контейнер
duration = min(trim_end - trim_start, 3.0)
bitrate = TARGET * 8 / duration * 0.92   # kbps, 8% запас на промах энкодера
fps = min(input_fps, 30)

for attempt in 1..6:
    pass1: ffmpeg -pass 1 -b:v {bitrate}k -deadline good -cpu-used 4 -f null
    pass2: ffmpeg -pass 2 -b:v {bitrate}k -minrate {b*0.5}k -maxrate {b*1.45}k
           -deadline good -cpu-used 1 -row-mt 1 -an out.webm
    size = filesize(out.webm)
    if size <= 256 KB: SUCCESS       # первая попытка успешна в ~90% случаев
    bitrate = bitrate * (TARGET / size) * 0.93
    if bitrate < 150 kbps:           # лестница деградации: качество битрейтом
        fps: 30 -> 24 -> 20 -> 15    #   уже не спасти — снижаем fps
        bitrate пересчитать заново
FAIL -> статус «не удалось уместить», показать финальный размер
```

- Первая попытка двухпроходного VP9 обычно попадает в лимит; итерации — страховка.
- Разрешение не снижается (512 — требование формата).
- После успеха: контрольная проверка размера в байтах и длительности через ffprobe.
- Пиксельный формат `yuv420p`, `-g 90` (одна keyframe-группа на 3 с — экономит размер).

## 5. UI: единое окно, файлы построчно

```
┌──────────────────────────────────────────────────────────────────────┐
│ [+ Файлы] [Вставить Ctrl+V] [Конвертировать всё] [Папка вывода: …]  │
├────────────┬──────────────────────────────┬──────────────────────────┤
│  ПРЕВЬЮ    │  ПАРАМЕТРЫ                   │  ВЫХОД                   │
├────────────┼──────────────────────────────┼──────────────────────────┤
│ [превью    │ Длительность: [0.0]–[3.0] с  │ ● Готово   248 КБ        │
│  ▶ плеер   │ ────█████████────  таймлайн  │ [▶ Превью] [Открыть]     │
│  0:01/0:03]│ Масштаб: [Растянуть ▾]       │ 4.2 МБ → 248 КБ          │
│ cat.mp4    │ Размер: [512]×[512]          │                          │
│ 4.2 МБ     │ Формат: [WebM VP9 ▾]         │                          │
├────────────┼──────────────────────────────┼──────────────────────────┤
│ [превью]   │ …                            │ ◌ В очереди / 45%        │
│ dog.gif    │                              │                          │
│ 1.1 МБ     │                              │                          │
└────────────┴──────────────────────────────┴──────────────────────────┘
```

**Загрузка файлов (пакетная):**
- Диалог выбора (мультивыбор) и drag&drop в окно.
- Буфер обмена (Ctrl+V или кнопка): список файлов (CF_HDROP) и растровое изображение (сохраняется во временный PNG и добавляется как строка).

**Строка файла:**
- Слева: превью-миниатюра; по клику — воспроизведение внутри строки с полосой прокрутки и текущим временем / общей длительностью. Имя файла и **размер входа**.
- Посередине: обрезка длительности двумя ползунками на таймлайне (по умолчанию 0–3 с; поле ввода с точностью 0.1 с), режим масштабирования, размеры (512×512), выходной формат.
- Справа: статус (в очереди / прогресс % / готово / ошибка), **размер выхода** и строка «вход → выход», превью результата (проигрывается в `<video>` — WebM играется нативно), кнопки «Открыть папку», «Повторить».

**Предпросмотр входных файлов:** MP4(H.264)/WebM/GIF/картинки играются в WebView нативно. Для MOV/AVI/MKV/HEVC — быстрый низкокачественный прокси (ffmpeg: 256px, `-crf 40 -cpu-used 8`, генерируется в фоне при добавлении файла).

**Очередь:** параллельно `max(1, ядра/2)` задач (VP9 тяжёлый, внутри `-row-mt`); остальные ждут. Конвертация по кнопке «Конвертировать всё» или по кнопке в строке.

**Выход:** папка вывода настраивается (по умолчанию рядом с исходником), имя `{имя}_sticker.webm`, при коллизии — суффикс `(1)`.

**Настройки** — `settings.json` рядом с exe (портативный режим), temp-файлы в `%TEMP%`, чистятся при выходе.

## 6. Этапы разработки

| Этап | Содержание | Критерий готовности |
|---|---|---|
| **M1. Ядро** | Rust-модуль: обёртка ffmpeg, алгоритм подгона размера (п. 4), обрезка, масштабирование. CLI для тестов | Набор эталонных файлов (длинное видео, 4K, GIF, фото, вертикальное видео) конвертируется в валидные ≤256 КБ / ≤3 с; автотесты проверяют байты и длительность |
| **M2. GUI Windows** | Окно со строками, диалог/drag&drop, параметры, очередь, прогресс, размеры вход/выход | Пакетная конвертация 10 файлов из UI |
| **M3. Полировка** | Буфер обмена, предпросмотр с таймлайном, прокси для экзотики, settings.json, обработка ошибок (битые файлы, нет места) | Портативная папка ≤35 МБ работает на чистой Win10/11 |
| **M4. Android** | Сборка Tauri под Android, ffmpeg в jniLibs, share-интент «Отправить в Стикер», SAF для выбора файлов | APK ≤ 40 МБ, паритет функций |

## 7. Структура проекта

```
sticker/
├─ src-tauri/            # Rust: очередь, ffmpeg-раннер, алгоритм подгона
│  ├─ src/core/          #   логика конвертации (без UI, тестируется отдельно)
│  └─ bin/win/ffmpeg.exe #   и ffprobe.exe (кастомная сборка)
├─ src/                  # UI: HTML/CSS/JS (без тяжёлых фреймворков — vanilla или Preact ~4 КБ)
└─ tests/fixtures/       # эталонные медиафайлы для автотестов
```
